PROJECT SCOPE
=============
Refactored best model checkpoint selection across three training scripts to use
Equal Error Rate (EER) as the primary metric and ROC-AUC as a tie-breaker,
replacing the previous F1-score based selection.

FILES MODIFIED
==============
1. train.py                    (CNN + Mel Spectrogram)
   - Lines changed: ~118 (50% of script)
   - Changes: Added EER computation, best model tracking, replaced end-of-
     training save with epoch-based save when EER improves

2. train_resnet.py             (ResNet18 + Mel Spectrogram)
   - Lines changed: ~68 (21% of script)
   - Changes: Updated validation function, changed scheduler mode, added 
     EER-based model selection

3. train_wav2vec.py            (Wav2Vec2 Model)
   - Lines changed: ~70 (19% of script)
   - Changes: Updated validation function, changed scheduler mode, added 
     EER-based model selection with early stopping

KEY CHANGES SUMMARY
===================

1. METRIC COMPUTATION
   - Added: ROC curve calculation from validation probabilities
   - Added: EER (Equal Error Rate) computation from ROC curve
   - Added: ROC-AUC calculation
   - Retained: F1-score (for reporting only, not model selection)

2. MODEL SELECTION LOGIC
   PRIMARY:   Save when EER decreases (lower is better)
   SECONDARY: Use higher ROC-AUC as tie-breaker when EER is equal
   
3. LEARNING RATE SCHEDULER
   Changed from 'max' mode (for F1) to 'min' mode (for EER)

4. VALIDATION FUNCTIONS
   train_resnet.py & train_wav2vec.py now return 5 values instead of 3:
   - Was: val_loss, val_acc, val_f1
   - Now: val_loss, val_acc, val_f1, val_eer, val_roc_auc

5. LOGGING
   Each epoch prints EER and ROC-AUC values
   Best model message shows both EER and ROC-AUC
   Final output shows Best EER and Best ROC-AUC

SCIENTIFIC RATIONALE
====================
EER is the standard metric for authentication/biometric systems (including
deepfake detection) because:
- It represents the threshold-independent optimal operating point
- It's where False Positive Rate = False Negative Rate
- It doesn't assume a specific decision threshold (unlike F1-score)
- ROC-AUC provides secondary evaluation when EER is similar

IMPACT ON MODEL SELECTION
=========================
The best model checkpoint selected may be from a DIFFERENT EPOCH than with
F1-score based selection. This is expected because:
- EER and F1 optimize for different objectives
- EER finds the threshold-independent optimal point
- F1 assumes a specific decision threshold
- For deepfake detection, EER is the more scientifically sound choice
